# 战略标签体系（6 个，可叠加）

## 使用规则

1. **多标签可叠加**：一款产品可能同时命中多个标签
2. **主标签写在最前**——选最能解释产品成功逻辑的那个
3. **辅助标签通过组合元素前缀挂载**——前缀来自封闭集合（吸量玩法/吸量题材/创新机制/玩法融合/轻量化/怀旧IP）
4. **写法统一**：使用本文档定义的标准名称，不要换说法（例如统一用"紫海战略"，不用"紫海赛道"）

## 判定流程

拿到一款新游戏时，按以下顺序判断（**优先级从高到低**，主标签从靠前命中的标签里选）：

1. **赛道定位**：是红海赛道里的差异化突围吗？→ 可能是 `紫海战略`
2. **赛道选择**：是公司战略持续投入的重点品类吗？→ 可能是 `重点赛道`
3. **玩法创新**：有绝对创新或组合创新吗？→ 可能是 `创新机制`
4. **机制融合**：是 2+ 成熟玩法融合吗？→ 可能是 `玩法融合`
5. **题材替换**：是基于已有爆款玩法套新题材/美术吗？→ 可能是 `题材美术换皮`
6. **平台迁移**：是把 PC / Steam / 端游 / 主机产品的核心玩法转化到手游 / 小游戏形态吗？→ 可能是 `端游转小游戏`（**优先级最低**，一般作为辅助标签使用，主标签优先从 1-5 中选）

每命中一个，就把它加入候选集；最后从候选集里选 1 个为主标签，其余通过组合元素前缀作为辅助挂载。

> **关于 `端游转小游戏` 的特殊说明**：该标签判定范围已放宽，覆盖 PC / Steam 独立游戏 / 端游 / 主机产品等任何重度形态向手游 / 小游戏的迁移。但由于"平台迁移"本身只解释来源，不解释成功逻辑（真正驱动成功的往往是创新机制或玩法融合），**该标签优先级放在最低**。当一款 Steam → 手游产品同时命中 `创新机制` 或 `玩法融合` 时，优先选后者作为主标签，`端游转小游戏` 不必强行写入标签行。

## 标签 1：紫海战略

**KSF 核心定义**

在已有的红海市场里，小体量游戏通过小众但潜力巨大的玩法空白点结合用户的痛爽点突围。"紫海"区别于纯粹红海（完全竞争）和纯粹蓝海（无竞争），是把红海中被遗忘的领域剥离出来"蓝化"形成自己的"紫海"。

**判定信号（命中任一即可考虑）**

- 在拥挤的赛道（SLG/MMO/RPG）里做差异化突围
- 跨圈层吸量（用一个圈层的题材吸引另一个圈层）
- 玩法轻量化降低门槛
- 单机化叙事（去社交压力）
- 轻社交 / 零社交
- 时间成本低（不强制每日上线、碎片化）
- 注重用受欢迎的题材吸量

**典型案例**

- 《无尽冬日》——通过"休闲外衣"降低初期 SLG 社交压力
- 《燕云十六声》——轻量化 MMO 体验
- 《球球大作战》——休闲竞技 + 创新机制

## 标签 2：重点赛道

**KSF 核心定义**

公司战略持续投入研究、征集和验证可立项机会的赛道。一般指有明确品类容量、可持续迭代的方向。

**判定信号**

- 公司战略性投入的赛道
- 持续研究、征集机会的方向（如轻度社交、动作 RPG、开放世界 RPG）
- 已有头部产品但赛道仍在演化

**典型案例**

- 《Monopoly Go》《猪来了》《超自然行动组》（轻度社交）
- 《白猫Project》《迷雾大陆》（动作 RPG）
- 《原神》《燕云十六声》（开放世界 RPG）

## 标签 3：端游转小游戏（优先级最低，一般作辅助标签）

**KSF 核心定义**

提取已在 PC / Steam / 端游 / 主机平台被验证过的核心玩法模式，将其转化为手游 / 小游戏形态，将成熟产品或 IP "降维"到轻度形态。

**判定范围（v2 放宽）**

覆盖以下任意一种来源平台 → 手游 / 小游戏 的迁移：

- 经典 PC 端游（如暗黑、魔兽、传奇）
- 主机产品（如塞尔达、暗黑破坏神）
- Steam 独立游戏 / 单机精品（如吸血鬼幸存者、Vampire Survivors、社畜大作战、缺氧、戴森球计划）

**优先级说明**

该标签只解释"来源平台"，不解释"成功逻辑"。真正驱动 Steam → 手游 / 端游 → 小游戏成功的核心因素，往往是 `创新机制` 或 `玩法融合` 或 `紫海战略` 等。**建议作为辅助说明，不作为主标签**；除非该产品的核心卖点就是"还原经典端游 IP 的情怀价值"（如《迷雾大陆》《英雄冒险团》），否则主标签优先从其它 5 个里选。

**判定信号（两类要点）**

① 还原体验
- 原平台核心玩法还原（保留原核心玩法，吸量老玩家）
- 美术还原与升级（题材美术使用吸量题材，运用轻度表达手法）
- 情怀营销（擦边老端游 / 经典 Steam 作品设计，引发情怀共鸣）

② 简化适配
- 玩法机制简化（保留原平台核心冲突点，针对小程序 / 手游特点简化）
- 游戏内容轻量化（玩法操作做减法，易用性便捷度做加法）

**典型案例**

- 《迷雾大陆》——还原暗黑核心玩法并简化（典型 PC 端游 → 小游戏）
- 《英雄冒险团》——还原魔兽核心玩法及题材并简化（典型 PC 端游 → 小游戏）
- 《弹壳特攻队》——Steam《吸血鬼幸存者》玩法迁移到手游（Steam → 手游，但主标签是 `创新机制`，本标签作辅助参考）

## 标签 4：创新机制

**KSF 核心定义**

分两类：

① **绝对创新**：需要绝对领先于市场的创意和技术（难度大、成本高、风险高）

② **组合创新**（稳定公式）：成熟玩法 + 乐趣玩法 + 变化适配
- 商业化框架成熟的品类：MMO、SLG、塔防、卡牌、放置、模拟经营
- 乐趣性玩法可来源于 steam 独立游戏或海外精品游戏

**判定信号**

- 引入了行业内未见过的核心机制 → 绝对创新
- 把成熟商业化框架与 steam 独立游戏的乐趣玩法组合 → 组合创新

**典型案例**

- 《逆水寒》《无尽冬日》《三国志战略版》《AFK》（组合创新代表作）
- 《无尽冬日》× 《冰气时代》（SLG + steam 独立游戏组合）
- 《快来当领主》× 《Lucky Defense》（卡牌 + 海外精品组合）

## 标签 5：玩法融合

> 注：原始名称是"玩法机制融合"，统一简写为"玩法融合"

**KSF 核心定义**

游戏内部组合两个及以上成熟玩法，形成差异化的玩法体验（如模拟经营+SLG）或丰富玩法内容（如三消+模拟经营）。

**玩法融合方法（必须遵守）**

- 不能 1 比 1 融合，必须区分主次玩法
- 融合的玩法之间需要互补性（轻重互补、长短线互补）
- "轻包重"是当前主流趋势（如《无尽冬日》：模拟经营轻 + SLG 重）

**两种融合模式**

① **核心玩法融合**
- 两类核心产品的玩法相融合形成一个组合紧密的核心玩法
- 玩法逻辑上具有一致性，轻度和轻度融合，重度和重度融合
- 玩法融合贯穿整个游戏，**无明显前后期玩法转换**
- 子类：轻度玩法融合、中重度融合

② **主次玩法融合**
- 多类玩法融合，保留一类或一组为核心玩法，其他为补充或轻度玩法
- 部分按前中后期阶段切换
- 子类：长线运营玩法补充、吸量转化轻包重、阶段性玩法融合

**典型案例**

- 《梦幻花园》——模拟经营 + 三消
- 《甜甜爱消除》——模拟经营 + 三消
- 《向僵尸开炮》——靶心 like + 肉鸽
- 《无尽冬日》——模拟经营转重度 SLG（吸量转化轻包重的代表）

## 标签 6：题材美术（换皮）

**KSF 核心定义**

基于已有的爆款玩法，快速套用新题材/美术，即换皮。

**三个要点**

① **主流玩法类型**：MMO / SLG / 模拟经营 是目前主流换皮的玩法类型

② **题材或美术先驱**：在同玩法游戏中做到题材、美术的先驱（首次使用某题材或美术手法）

③ **题材吸量考虑**：各种"皮"需要考虑题材吸量程度，优先考虑用户盘大、用户付费能力强的题材
- 西游
- 修仙
- 国风水墨
- 三国

**典型案例**

- 《向僵尸开炮》——靶心 like 中首次使用僵尸题材
- 《无尽冬日》——SLG 类中首次使用末日冰雪题材

## 组合元素前缀（封闭集合）

写"成功逻辑"标签行时，辅助轴的前缀**只能**从下列集合中选择，**禁止自造**：

| 前缀 | 用法 | 示例 |
|---|---|---|
| `吸量玩法` | 该轴突出的是某种玩法本身的吸量能力 | `吸量玩法【搜打撤】` |
| `吸量题材` | 该轴突出的是某种题材/美术的吸量能力 | `吸量题材【房车×动物拟人】` |
| `创新机制` | 该轴是某种创新机制（对应标签 4）| `创新机制【肉鸽】` |
| `玩法融合` | 该轴本身是若干玩法的融合（对应标签 5）| `玩法融合【模拟经营 × 弹幕射击】` |
| `轻量化` | 该轴的差异化是对原品类的轻量化处理 | `+ 轻量化` |
| `怀旧IP` | 该轴使用了怀旧 IP 作为吸量手段 | `怀旧IP【古墓丽影】` |

**组合规则**

- 最少 1 轴，最多 3 轴
- 多轴用 ` + ` 连接
- 每轴的具体内容用 `【】` 包裹
