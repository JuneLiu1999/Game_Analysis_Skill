# 语言调性规则

## 核心原则

报告语言追求**信息密度极高、客观冷静、可量化、结构化**。读者是行业内人士，不需要任何背景铺垫，每个字都要有信息价值。

---

## 一、人称与立场

- ❌ 禁止第一人称："我""我们""我认为""我们看到"
- ❌ 禁止主观判断词："好""坏""厉害""牛"
- ✅ 全程客观陈述，把判断藏在因果结构里

**反例**：我们认为这款游戏在搜打撤赛道很有潜力
**正例**：搜打撤赛道 2025 年累计下载 1 亿（同比增长 174%），《无限轮回》上线稳定微小畅销榜 Top20

---

## 二、句式结构

### 必用：因果结构

每句的本质必须是 `机制 → 效果` 或 `现象 → 判断`，禁止纯描述性句子。

**反例**：游戏中有肉鸽机制和搜打撤玩法
**正例**：通过肉鸽强化与搜打撤决策驱动玩家在"继续深入"与"及时撤离"之间形成反复博弈，建立"探索→撤离→构筑→成长→再探索"的持续循环

### 高频句式（鼓励使用）

- `形成 X 的 Y`
- `建立 X 的 Y`
- `构建 X 的 Y`
- `围绕 X + Y 设计 Z`
- `通过 X，驱动玩家 Y`
- `结合 X 与 Y，放大 Z`
- `在 X 过程中通过 Y 放大 Z`
- `推动玩家持续 X，在 Y 的多目标驱动下形成 Z`

---

## 三、词汇规范

### 禁用词汇

| 类别 | 禁用词 | 原因 |
|---|---|---|
| 程度副词 | 非常 / 极其 / 特别 / 相当 / 十分 / 很 | 主观、信息密度低 |
| 模糊词 | 大概 / 差不多 / 应该 / 可能（除真不确定外） | 削弱判断 |
| 口语化 | 其实 / 感觉 / 挺 / 还行 / 蛮 | 不够正式 |
| 情绪词 | 太棒了 / 厉害 / 牛 / 离谱 | 主观 |

### 标点禁用

- ❌ 感叹号
- ❌ 修辞疑问句（"难道不是吗？"）
- ❌ 省略号（除非引用原文）

---

## 四、符号系统（必须严格使用）

| 符号 | 用法 | 示例 |
|---|---|---|
| `【】` | 包裹关键标签 / 小标题 / 主标签 / 子模块标题 | `【紫海战略】` `【玩法赛道分析：搜打撤】` |
| `《》` | 包裹游戏名、产品名、IP 名 | `《无限轮回》` `《古墓丽影》` |
| `→` | 描述循环路径、闭环链 | `探索→撤离→构筑→成长→再探索` |
| `●` | 二级 bullet（仅用于核心乐趣/核心目标） | `● 核心乐趣：...` |
| `①②③` | 段内序号（吸量、付费、留存阶段） | `① 题材吸量` |
| `+` | 连接组合元素 | `紫海战略 + 怀旧IP【古墓丽影】` |
| `×` | 连接交叉融合的两个维度 | `房车 × 动物拟人` `模拟经营 × 弹幕射击` |

---

## 五、量化表达

### 规则

凡是涉及"赛道大小""产品成绩""题材热度"的判断，必须配量化证据。

### 标准量化指标

- **赛道**：年度累计下载 / 累计收入 / YoY 增长率
- **产品**：累计下载 / 累计收入 / RPD / 榜单位置（畅销榜/小程序榜/iOS 榜+Top N）
- **留存**：次留 / 7 留 / 30 留 / 60 留（百分比）
- **题材热度**：相关视频播放量 / 话题播放量 / 系列累计销量 / 博主关注数

### 写法示范

✅ `《古墓丽影》系列游戏累计销量突破 1 亿份`
✅ `海外：Youtube 房车旅行博主关注 770 万，单视频播放破 2 亿`
✅ `《无尽冬日》累计下载 1.6 亿`
✅ `次留 29%；7 留 12%；30 留 3%`

❌ "这款游戏很受欢迎"
❌ "题材很有热度"
❌ "下载量不错"

## 六、专业术语统一表

下列术语在报告中统一写法：

| 标准写法 | 不要使用 |
|---|---|
| 紫海战略 | 紫海赛道、紫海定位 |
| 玩法融合 | 玩法机制融合 |
| 题材美术（换皮）| 换皮 / 题材替换 / 套皮 |
| 搜打撤 | 撤离类 / extraction |
| 肉鸽 | roguelike / Rogue |
| Build / 构筑 | 配装 / 流派搭配（仅在描述具体流派时用） |
| 局内 / 局外 | 内场 / 外场 |
| 核心乐趣 / 核心目标 | 玩法体验 / 玩家目标 |
| 长期主义 | 长线设计 / 长期规划（用于成功逻辑副标题，固定）|
| 闭环 | 完整循环 / 全链路 |
| RPD | ARPU / 人均付费 |

## 七、段落组织规则

### 列项规则

- 用 `①②③` 标注同级要点
- 多于 3 项时考虑拆分子标题
- ① 和 ② 之间空一行

### 加粗与高亮

- 报告全文**禁用加粗**（节省视觉噪声，所有关键信息靠 `【】` 突出）
- **不使用** Markdown 的 `**bold**`、`#标题`、`> 引用` 等

### 段落长度

- 单段不超过 4 行
- 信息密度高的判断句独立成段

---

## 八、整篇风格自检

输出前问自己 5 个问题：

1. 全文有出现"我""我们"吗？→ 删
2. 有无量化证据的判断句吗？→ 加数据
3. 有任何形容词冗余吗（"非常""极其""挺"）？→ 删
4. 每段是否能在因果链上闭环？→ 不闭环就重写
5. 符号使用是否一致（【】《》→●）？→ 统一

5 个问题全过，才算合格。
