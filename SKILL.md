# 游戏市场潜力分析（SKILL）

## 一、写作前必读

### 1. 强制确认输入

开始写作前必须确认以下信息齐全。**任何一项缺失就主动询问用户，禁止凭空编造数据**：

- 待分析游戏名称（或概念描述）
- 玩法/赛道分类（如动作RPG、SLG、搜打撤等）
- 主要题材/IP（如有）
- **对标产品**（用户可直接指定；未指定时按下方"对标产品获取流程"处理）
- 对标产品的关键数据：榜单位置、留存（次留/7留/30留/60留）、下载/收入/RPD（拿不到的可标"暂无数据"）

#### 对标产品获取流程（强制）

1. **用户已指定对标**：直接使用用户给的对标产品，不再额外推荐。
2. **用户未指定对标**：必须先停下来，给用户**推荐 2–3 个最匹配的候选对标**，列出每个候选的核心匹配理由与已知关键数据；**等用户选定后再开始写报告**，禁止自行选定对标直接生成。

#### 对标产品的相似性维度（推荐对标时必须逐项匹配）

推荐候选对标时，要逐项评估以下维度并在推荐说明里明示匹配/差异点。**任何一个维度差异较大都必须在推荐时明确标注，不能掩盖**：

- **核心玩法机制**：是否同一玩法核心（如 Roguelite、塔防、SLG、模拟经营）
- **玩法形态/视角**：2D / 2.5D / 3D；俯视角 / 侧视 / 第一人称 / 第三人称（FPS / TPS / 卡牌等）—— **视角与维度不同直接判为相似度低**
- **战斗形态**：弹幕射击 / TPS 射击 / 自动战斗 / 回合制 / 即时战斗 等
- **题材/美术**：主要题材范畴（科幻/末日/二次元/Q版/写实/卡通）
- **平台与体量**：手游 / 小游戏 / Steam / 主机；轻度 / 中度 / 重度
- **商业模式**：买断 / 内购 / 混合

**反例**（绝对禁止）：把 2D 弹幕射击 Roguelite 拿去对标 3D TPS Roguelite，仅因为同属"Roguelite + 射击"就认定为合理对标——这种对标只在赛道宏观数据层面成立，不能用于留存/付费的具体数据迁移。

#### 对标产品的三种合法形态（优先级从高到低）

1. **直接对标**：被分析产品本身（如已上线手游/小游戏，直接用自身数据）
2. **概念对标**：同款游戏在不同平台的版本（如 Steam 原作分析手游化潜力时，用 Steam 端的成绩 + 同品类成功手游化案例的留存/付费数据组合）
3. **赛道对标**：同品类已成功的手游/小游戏（用于全新概念分析、或被分析产品没有任何上线数据时；必须明确标注"赛道对标"以提醒读者这是 proxy 数据）

无论用哪种对标，留存数据栏的头部必须明确标注对标产品名，避免读者误以为是被分析产品自身数据。

### 2. 流程总览

按以下顺序产出：
1. 选战略标签（多标签可叠加，见 `reference/战略标签体系.md`）
2. 写成功逻辑（总论）
3. 写吸量（论据 1：进入动机）
4. 写留存（论据 2：持续动机）
5. 写付费（论据 3：变现能力）
6. 跑"输出前自查清单"（本文末尾）

### 3. 必读参考文件

- `reference/战略标签体系.md`——6 个标签的判定标准、典型信号、案例
- `reference/语言调性规则.md`——句式、词汇、符号、禁用词
- `examples/`——4 份完整黄金样本，写作前必须扫一遍
- `templates/空白报告模板.md`——直接复制使用

## 二、整体骨架（4 部分固定顺序，不可调换）

| 顺序 | 部分 | 定位 |
|---|---|---|
| 1 | 成功逻辑 | 总论 / 结论层（一段话回答"为什么这游戏能打"）|
| 2 | 吸量 | 进入动机的论据（赛道+差异化）|
| 3 | 留存 | 持续动机的论据（前/中/后期循环）|
| 4 | 付费 | 变现能力的论据（核心+长期）|

## 三、各部分写作规范

### 3.1 成功逻辑

**结构（两段固定）**

第一段：标签行公式
```
成功逻辑：【主标签】[玩法/赛道名] + [组合元素前缀]【具体内容】[+ 可选第三轴] 版《对标产品》
```

- **主标签**：从战略标签体系里选一个为主（紫海战略 / 重点赛道 / 端游转小游戏 / 创新机制 / 玩法融合 / 题材美术换皮）
- **组合元素前缀**：必须从下列**封闭集合**里选，禁止自造：
  - `吸量玩法`
  - `吸量题材`
  - `创新机制`
  - `玩法融合`
  - `轻量化`
  - `怀旧IP`
- 可叠加 2-3 个轴（最多 3 轴），用 ` + ` 连接
- 末尾必须写 `版《对标产品》`

第二段：长期主义段
```
长期主义：前期通过X建立Y，中期以X放大Y，后期结合X构建Y，形成"A→B→C→...→再A"的持续回流与变现闭环
```

- 副标题"长期主义："**固定**，不可替换
- 三段式（前期/中期/后期）必须齐
- 句式骨架：
  - 前期 = 建立基础循环 / 建立高频循环 / 建立收益预期
  - 中期 = 放大策略深度 / 放大重复动机 / 形成差异化
  - 后期 = 构建长线目标 / 构建付费承接 / 构建闭环
- 收尾的箭头链非强制但**强烈推荐**——能写出"探索→撤离→构筑→成长→再探索"这种闭环最好

### 3.2 吸量

**结构（两个子模块固定）**

```
【玩法赛道分析：赛道名】
① 赛道总体数据：[三选一形态]
② 头部产品数据：[可选]

【差异化分析：X】
① 题材吸量 / 玩法吸量：[判断句]
[量化证据]
② 题材吸量 / 玩法吸量：[判断句][可选]
[量化证据]
```

**关键规则**

- 两个 `【】` 子标题必须出现：`【玩法赛道分析：X】` 和 `【差异化分析：X】`
- 赛道总体数据三种合法形态，按数据可得性选用：
  1. 量化型：`移动端【X】赛道2025年累计下载 N（同比 X%），累计收入 $N（同比 X%）`
  2. 头部产品罗列：`《产品A》累计下载/收入...；《产品B》累计下载/收入...`
  3. 定性型：`赛道总结：移动端【X】赛道用户盘子大、付费能力强`
- 差异化分析的轴可灵活组合：`题材+玩法` / `玩法1+玩法2` / `题材1+题材2` / 单轴
- 每个差异化点必须配**量化证据**：销量、播放量、榜单名次、累计下载/收入等。**禁止纯定性断言**
- 单一轴不突出时，可省略 ②

**论据来源硬规则（v2 新增，每写一个差异化点都必须遵守）**

- **玩法吸量** 论据 = 论证「**该类型/同赛道游戏整体被验证成功**」，必须列举 ≥2 款**同类型其它产品**的下载/收入/播放量等数据；**严禁把"被分析产品自身的成绩"当作玩法吸量论据**——被分析产品自己赚了多少钱属于"结果"，不是"为什么这类玩法吸量"的"原因"
- **题材吸量** 论据 = 论证「**该题材在内容生态上自带流量**」，必须列举短视频/社媒话题播放量、热搜次数、UGC 数量、或同题材其它产品的吸量证据；**严禁仅靠"题材听起来很有共鸣"做定性断言**
- 论据中**避免循环论证**：若被分析产品本身就是该赛道头部，可在「头部产品数据」节列其数据，但**不能**把它用作"差异化分析→玩法吸量"的核心论据
- 若同类型同视角同形态的成功产品确实稀缺（如 3D TPS Roguelite 手游样本少），允许在论据里**分两层论证**：(a) 大赛道（如 Roguelite 射击整体）已验证成功；(b) 同视角/同形态产品在 Steam 等其它平台已验证成功——但需明确标注两层范围，不可混为一谈

### 3.3 留存

**结构**

```
【留存对标产品《X》：次留 N%；7留 N%；30留 N%[；60留 N%]】
[或：【留存对标产品《X》ST暂无留存数据】]

① 前期：[机制1]+[机制2]+[机制3]，[效果描述]
● 核心乐趣：[体验层描述，必须覆盖局内+局外]
● 核心目标：[循环与成长描述，必须包含一条箭头链]

② 中期：[机制1]+[机制2]+[机制3]，[效果描述]
● 核心乐趣：...
● 核心目标：...

③ 后期：[机制1]+[机制2]+[机制3]，[效果描述]
● 核心乐趣：...
● 核心目标：...
```

**关键规则**

- 头部留存数据三围（次留/7留/30留）必须齐；有 60 留就加上；查不到就写"ST暂无留存数据"
- 三段式 ① 前期 / ② 中期 / ③ 后期 **不可缺**（"长期"和"后期"可互换，但全文统一）
- 阶段标题格式：`阶段：[3 个机制元素拼接]，[效果描述]`
- 每段内必须有 **`● 核心乐趣` + `● 核心目标`** 双栏
  - **核心乐趣** = 体验层（玩家感受到什么，必须描述局内+局外两条线）
  - **核心目标** = 循环成长层（达到什么目标，循环如何形成）
- 核心目标段末必须包含一条**箭头链**描述循环路径，例如：
  - `"探索→获取资源→构筑选择→风险决策→撤离→养成强化→轮回再进入"`
  - `"驾驶房车→探索开图→种植生产→订单交付→收益扩展→再出发"`
- 阶段演进逻辑（必须遵守）：
  - 前期 = 建立基础循环 + 解锁核心机制认知
  - 中期 = 多线程目标并行 + 构筑/Build 分化
  - 后期 = 多目标驱动长期回流 + 赛季/排名/养成深化

### 3.4 付费

**结构**

```
【付费对标产品《X》：[榜单位置 或 累计下载/收入/RPD 数据]】
① 核心付费点：[绑定核心玩法机制]
[可选：因果驱动描述]
② 长期付费点：[绑定赛季制+养成坑深+通行证]
[可选：因果驱动描述]
```

**关键规则**

- 头部数据二选一：榜单位置（如"小程序畅销榜Top43"）OR 累计下载/收入/RPD 三件套
- 固定两层：① 核心付费点 + ② 长期付费点。**不允许**新增第三层
- 核心付费点必须**绑定该游戏的玩法机制本身**（如死亡惩罚→撤离收益、骰子→骰子种类付费、塔防→挑战次数与复活）
- 长期付费点几乎固定包含：`赛季通行证 + 角色/装备养成坑深 + 外观皮肤/装饰`
- 两种合法密度：
  - 简洁版：`核心付费点：A + B + C + D`（清单式）
  - 详细版：`围绕 X，结合 Y 设计 Z，驱动玩家在关键节点进行付费选择`（带因果）
- 视分析深度选用，复杂/旗舰产品建议详细版

## 四、语言调性硬规则

详见 `reference/语言调性规则.md`，核心铁律：

1. **无第一人称**——不出现"我""我们""我认为"
2. **名词密度极高**——少用形容词、副词、修饰语
3. **每句必为因果结构**——"机制→效果" 或 "现象→判断"，禁止纯描述
4. **量化优先**——能用数据就用数据
5. **符号系统**：
   - `【】` 包裹关键标签 / 小标题
   - `→` 描述循环路径
   - `●` 二级 bullet（核心乐趣/核心目标）
   - `①②③` 段内序号
6. **禁用词**：
   - 程度副词："非常""极其""特别""相当"
   - 情绪词：感叹号、修辞疑问
   - 口语化："其实""感觉""差不多""挺"
   - 不确定词：除非确实有判断模糊
7. **高频句式**（鼓励使用）：
   - `形成 X 的 Y`
   - `建立 X 的 Y`
   - `构建 X 的 Y`
   - `围绕 X + Y 设计 Z`
   - `通过 X，驱动玩家 Y`

## 五、输出前自查清单（每篇必跑）

输出报告前，逐项确认：

**结构层**
- [ ] 4 部分齐全且按 `成功逻辑→吸量→留存→付费` 顺序
- [ ] 成功逻辑包含标签行 + 长期主义段两段
- [ ] 吸量包含 `【玩法赛道分析】` + `【差异化分析】` 两个子模块
- [ ] 留存有三阶段，每阶段均含 `● 核心乐趣` + `● 核心目标`
- [ ] 付费严格两层：`① 核心付费点` + `② 长期付费点`

**内容层**
- [ ] 主标签来自 6 个标签封闭集合
- [ ] 组合元素前缀来自封闭集合（吸量玩法/吸量题材/创新机制/玩法融合/轻量化/怀旧IP）
- [ ] 留存三围数据齐全（或明确标"暂无"）
- [ ] 每个差异化点都有量化证据
- [ ] **玩法吸量论据 ≥2 款同类型其它产品的数据，未把被分析产品自身当论据**
- [ ] **题材吸量论据含话题播放量/热搜/同题材产品数据，非纯定性断言**
- [ ] **对标产品在「核心玩法 + 视角维度（2D/3D/FPS/TPS）+ 战斗形态」三项上至少匹配 2 项**，差异点已在文中明示
- [ ] 留存核心目标段含箭头链
- [ ] 付费核心付费点绑定该游戏玩法机制（不是泛泛而谈）

**语言层**
- [ ] 无第一人称
- [ ] 无禁用词（非常/极其/感觉/其实/...）
- [ ] 无感叹号、无修辞疑问
- [ ] 符号使用规范（【】→●①②③）
- [ ] 每句基本是因果结构

**全任意一项不通过，必须修订后再交付。**
